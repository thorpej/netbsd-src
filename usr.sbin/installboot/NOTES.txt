== 1 ==
Add new "board" option.  "board" takes a board name like "bananapi-r2"
or "orangepi-zero-plus".  This provides defaults for the path to the
u-boot loader as well as the SoC type.  The name mappings come from
a plist file, that also provides a nice descriptive name of the board.

== 2 ==
Add new "soc" option.  This selects the SoC-specific method for writing
u-boot to the media.

== 3 ==
Allow the path to the primary bootstrap to be optional.  If not specified,
inferred from "board" option (/usr/pkg/share/u-boot/<board>).

== 4 ==
Allow the path to the primary bootstrap to be a directory rather than
just a regular file.

== 5 ==
Add new "media" option.  This optionally selects which u-boot loader to
write to the media, in an SoC/board-specific way, if, for example, the
SoC requires different u-boot images for sdmmc vs emmc media.


Board / SoC write methods are selected in a more-to-less-specific fashion:

	-- First search for a method corresponding to <soc>-<board>
	   (e.g. sunxi-orangepi-zero-plus).

	-- If not found, search just by <soc> (e.g. sunxi).


SoC plist schema:

Top-level:
	Dictionary, keyed by SoC type.  Value is a SoC object dictionary.

SoC object dictionary:
	Key: "u-boot-methods", value: u-boot methods dictionary.

U-boot methods dictionary:
	Key: method type (default method keyed "default").  Value is a
	u-boot method object.

U-boot method object: An array of u-boot operation objects.

U-boot operation object:
	Key: filename - string with the blob file name (required)
	Key: file-size - integer with the max bytes to copy from file
	     (optional) -- not present means "copy entire file"
	Key: file-offset - integer with the offset in file to start
	     copying from -- not present means "from start of file"
	Key: image-offset - integer with the offset into the disk image
	     where to write the file -- not present means "offset 0".
	Key: preserve -- bool indicating that a partial sector should
	     have non-written contents preserved if true. (optional)

Example:

<plist>
<dict>
	<key>meson-gxl</key>
	<dict>
		<key>u-boot-methods</key>
		<dict>
			<key>sd</key>
			<array>
				<dict>
					<key>filename</key>
					<string>u-boot.bin.sd.bin</string>
					<key>file-size</key>
					<integer>442</integer>
					<key>preserve</key>
					<true/>
				</dict>
				<dict>
					<key>filename</key>
					<string>u-boot.bin.sd.bin</string>
					<key>file-offset<key>
					<integer>512</integer>
					<key>image-offset</key>
					<integer>512</integer>
				</dict>
			</array>
			<!-- Presumably there's an emmc method. -->
			<!-- ...and a usb method. -->
		</dict>
		<key>boards</key>
		<dict>
			<-- Board objects loaded at run-time. -->
			<-- Key is equal to board-name property of
			    board object. -->
		</dict>
	</dict>
	<key>sunxi</key>
	<dict>
		<key>u-boot-methods</key>
		<dict>
			<key>default</key>
			<array>
				<dict>
					<key>filename</key>
					<string>u-boot-sunxi-with-spl.bin</string>
					<key>image-offset</key>
					<integer>8192</integer>
				</dict>
			</array>
		</dict>
		<key>boards</key>
		<dict>
			<-- Board objects loaded at run-time. -->
			<-- Key is equal to board-name property of
			    board object. -->
		</dict>
	</dict>
</dict>
</plist>
